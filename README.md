I'm a **DevOps Engineer** currently at **Aguken AI**, where I own the infrastructure for a live AI voice platform — from Kubernetes cluster operations on AWS EKS to GitOps delivery pipelines and full-stack observability. I care about automation that is reliable, infrastructure that scales without drama, and deployments that developers can trust.

## Featured Projects

### 🏭 EKS Microservices Platform *(Production — Aguken AI)*
Running 10+ microservices on AWS EKS with Istio for traffic management, Argo CD for GitOps delivery, and a Prometheus + Grafana observability stack. Reduced deployment cycle time by 70% and cut compute spend by ~30% through systematic right-sizing.
`EKS` · `Istio` · `Argo CD` · `Prometheus` · `Grafana` · `Terraform` · `Helm`

---

### 🚀 [Chattingo — Containerized CI/CD Deployment](https://github.com/PragyaLaxkar/chattingo)
Forked a full-stack React + Spring Boot + WebSocket chat application and built the complete DevOps layer from scratch: multi-stage Dockerfiles for frontend and backend, a root-level `docker-compose.yml` for local orchestration, and a Jenkins pipeline covering Git clone → image build → filesystem security scan → image vulnerability scan → Docker Hub push → VPS deploy. SSL via Let's Encrypt on Hostinger VPS.
`Docker` · `Jenkins` · `Multi-stage builds` · `Nginx` · `VPS` · `CI/CD`

---

### 🐳 [Docker Automation Script](https://github.com/PragyaLaxkar/Docker-Automation-Script)
A menu-driven Bash utility that wraps 10+ Docker operations (pull, run, start, attach, stop, inspect, remove) into a clean interactive interface. Built to reduce repetitive CLI friction in daily container workflows.
`Bash` · `Docker` · `Shell scripting`

---

### ⚙️ [Multi-Service Automation Platform](https://github.com/PragyaLaxkar/Multi-service-automation-platform)
A Python-based web automation platform with a CGI backend and HTML frontend, designed to automate and orchestrate multi-service workflows from a browser interface.
`Python` · `CGI` · `Automation` · `Web interface`
